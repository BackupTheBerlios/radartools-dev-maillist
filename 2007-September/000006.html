ok, i played a bit around and put some support for plugins  inside of
rat. it was a nice change/relaxation after reading a lot of theory in
the last few weeks.
i already uploaded it, and also added two example plugins (which we
can delete later, for the release). so, after an update you will see
it working.
In general:
- plugins should be put into rat/plugins/ or .rat/plugins/ directories.
- the name should be "rat_plugin_{MYNAME}.sav" or "rat_plugin_{MYNAME}.pro"
- the included module(procedure) should be called "rat_plugin_{MYNAME}"
- it should be able to return a parameter "PLUGIN_INFO", which should
have the structure: {menu_name:"The name of the plugin as it should
appear in the menues", menu_pos:["positions inside the menu","at the
moment, only the menu-roots can be given, as sar, polsar,polinsar,
etc."]
- the menu_name-call will appear in all sub-menus, as specified in
menu_pos. Additionally, all plugins will be available in "General->All
Plugins->". (For example, test-plugin will appear in "polinsar->Extern
plugins->", and the quote-plugin will appear in "polinsar->Ext. pl->"
and in "help->Ext. pl->". Both of them will appear in "general->All
plugins->..")
- for the moment, to make it easier, I programed it so that the
plugins menues appear at the top of the sub-menues. if there will be
the need and the time, one can also describe the positions more
precise.

but just check it out, the description is more complicated than the
actual usage.
the plugin-file for quotes contains more info about plugin handling,
which should later be transported to tex and the rat-guide.
it's a proposal for this given task, and can be of course modified.
