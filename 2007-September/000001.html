Hi rat-developers,

after some longer unplanned breaks I'm now finally back in the uni,
and for restarting the work, I merged the 0.19er brunch into the
trunk. And, actually, there was almost nothing to merge, since Anderl
did already the whole job. But now everything is completely clean and
ready for new modules.

For Infos/Questions:
- I added some functions into rat_install to modify the "rat" script
in the rat directory to adjust it to the used directory. However,
since the file "rat" will be different in all individual installations
(mine, yours, etc.), it will force an update of the svn repository
every time. Therefore I think, it is ok to remove it from the
repository, since it will be generated automatically during
rat_install (and only for the unix-platforms, so that windows users
will not have it all). If you think, it's not ok like this or you have
a better strategy, let me know, and we'll modify it.
- Concerning PolInSAR, RAT seems to work fine and for the moment I
don't have any ideas for improvements (except perhaps the coherence
bias removal functions).
- I want also to finish the bash--only interface, with the possibility
to write rat-scripts, without any graphical outputs. For the moment it
"mostly" works (only with some specific functions), but it needs some
checks and some clean guidelines for writing new modules. And one
tutorial and a reference are really necessary.
- Else: in the near future I want to concentrate a bit more on my
thesis and really to try not to start to add new functionality into
rat. But let's see...

Poka,
Maxim
