Am Freitag 30 November 2007 schrieb Maxim Neumann:
> Hi all,
> how is it if we include a small $main$ program at the end of rat.pro
> (the last two lines in the now-checked-in version), so that whenever
> one uses
> ".r rat" or ".run rat",
> the program gets compiled and started automatically.
> That is, one wouldn't need anymore to type explicitly "rat" in the
> shell to run Rat.
> If one wants only to compile everything, one can use ".compile rat".
> With this we would use .run and .compile as initially intended by IDL,
> to "compile and run", and to "compile" only.
> I can see the rightful criticism, since it is against the habits.
> On the other hand, if one gets used to it, in the long run one can
> save a lot of explicit calls of rat.
>
> Greetings,
> Maxim
>
> p.s.: one will need to modify the rat-script by changing ".r rat" to
> ".compile rat" (it will happen automatically during a rat_install)

> can switch back, if it's unpractical
