Hi All,

This brings me to a very interesting topic regarding RAT functionality
and something I would love to work on. There seems to be some ideas
about data management in RAT already, this can be extended to include
layers. In addition for multitemporal sets RAT will greatly benefit with
some spatial/geographic metadata storage per file. This way scene
extents can be stored in some sort of auxillary RAT file and subsets
extracted from .rat data file with offsets calculated using geographical
extents. Much like the ENVI layers and .hdr file system. There also does
not seem to be a datatype associated with Multi-Temporal sets in the
Edit header section.

Andreas is the student you mentioned on the list and
available/interested in working further ?
