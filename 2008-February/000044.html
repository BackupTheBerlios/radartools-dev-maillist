We should start a discussion on these points!

The data managment stuff might be indeed a good way to implement multiple file 
handling. Originally it was written to simplify the jumping between many 
different results/decompositions/etc. of a single data set. But why not using 
it in a similar way for multi-files? I think it should be possible to do 
this. Maybe we should ask Stephane if he is interested in helping a bit here 
(he implemented the data handling)?

An auxiliary RAT file is already existing, the *.rit files you see. They are a 
possible place to store geographic information. Exporting to a common format 
would be of course necessary, too (ENVI .hdr might be a good choice).

>There also does not seem to be a datatype associated with Multi-Temporal sets 
>in the Edit header section.

No, because each file keeps his orignial type when using multifiles. This was 
the easier solution, otherwise you might need a multifile version for many 
different types (think about polarimetric multifiles, etc.)
