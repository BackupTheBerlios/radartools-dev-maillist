Hi Tisham,

The multitemporal stuff was implemented by a student and the code is not
so good. However, while working with him we had some nice discussions on
the topic. One point was to question how to store the data: The problem
is that a multitemporal data set in one might get horribly large. Just
imagine 10 full Envisat SLC scenes = roughly 10GB. Secondly, one might
want to add an additional file later (once it has been acquired).
Finally, how to hande multitemporal polarimetric data (now available
with TerraSAR, ALOS and Radarsat-2).

Therefore I think before we start implementing something better (which
would be nice and necessary), we should think about a good concept. In
fact, up to now I don't really know how to solve the problem. Somehow,
RAT needs to be modified in a way that it can load multiple layer, but
that is not an easy task... Does anybody have a good idea?

Cheers,
Anderl
