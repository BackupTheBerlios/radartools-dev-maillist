Hi Andreas and All,

the "save_image.pro" I take on my account. I added channel_defaults
just a few days ago.
My problem was in connection with a script, which used Rat to generate
images of some rat files (or all in the directory), as well as of all
sub-tracks inside an MB-data set. And without the channel_defaults it
didn't update the channels when I used the extract-functions from the
shell.
Anyway, now I got the point, why it shouldn't be there. I'll remove it
(if nobody did it already), and I'll find (later) another way to get
my images without conflicts.

Concerning rrat, i want to mention that I added the feature to check
automatically for the ".rat" file endings. This is for the people who
prefer to save the time to write this ending explicitly.

Concerning the rat common block in rrat.pro: has to be removed, of
course. Just wanted to mention than in the given style it contains an
error source. With "common rat, config" one will not get the config
structure, but the first structure of the common block "rat", which is
"types". But it will be called "config" and will lead only to
confusion. The best way to use common blocks, is e.g. just to write
"common rat" without any other names, and one will automatically get
all the structures as defined previously with their defined names.

Regards,
Maxim
