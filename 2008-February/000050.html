Hi all,

as it seems, it is called only from the following three routines:
- EM classification (polar & polin)
- Lee category preserving classification
- Simulated  Annealing

in all of them it is used together with block_trace over the
covariance matrices, so that it does not matter, either the results of
block_mm are floats or complex. But we should provide a keyword to
block_trace for giving back the real_part of result for some occasions
like the traces over covariance matrices (and use it in the previously
named routines). I can take care of these routines.

Regards,
Maxim

ps: you can check the routine /general/mm/mm_mm.pro which does the
same as block_mm, but in mathematical matrix notation (i.e. the result
is transposed compared to block_mm). I prefer to use this one, but
this is a matter of personal preferences.
