I also agree on everything as you, Anderl and Steph, said:
+ more docu + some standards + simplification of module writing
(object+procedural) + radical changes at first into /tags
This all is of course reasonable and I will also try to contribute
some parts, as the time will allow.

An update to plugins:
- I put the plugins into an extra svn-brunch
- additional required keyword for plugins: "/help", so that it is
possible to obtain some info on the plugin from the help-menu. For
more details, see the example plugin "rat_plugin_quote.pro".
- therefore, if you are writing a plugin, you MUST serve appropriately
these two keywords "HELP", and "PLUGIN_INFO".
