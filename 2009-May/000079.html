Hi all,

I really like this idea of a wiki (and use wiki it for some other projects 
quite intensively). It seems that you need a berlios account to modify 
content. This is good as this will avoid spam. And bad, as it will limit the 
user base contributing. But I think it is the only practical way. Without 
checking it, there should be some possibility to limit the rights to the 
documentation, i.e. having code developers and docu developers.

Anyway, I think we should move the RAT homepage to Berlios (I haven't found 
the time up to now), so why not using their nice infrastructure a bit more 
extensively...

Regards,
Andreas
