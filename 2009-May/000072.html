Hi Andreas,

the implementation of the rat batch mode was mere a fast hack, in order to
be able to use some scripts. It only hides the GUI, and disables some
preview functions for the data (which saves a lot of time on some lame
computers, like mine). But the whole GUI is still generated (and needs X),
and one can always show it with the command "show_gui" (or something
similar).

In order to provide a real batch-mode requires some more work. I think, it
shouldn't be too complicated, but one needs then to thoroughly separate the
gui and visual functions from other functionality (in some functions it's
too much interconnected).

Regards,
Maxim
